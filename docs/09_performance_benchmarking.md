# Manufacturing Process Copilot
## Document 09 — Performance Benchmarking

## 1. Dataset

All models were trained and evaluated on synthetic production order data generated by the factory simulation (`ml/scripts/generate_synthetic_data.py`).

| Split | Rows | Period | Purpose |
|---|---|---|---|
| Train | ~3,050 | Days 1–385 (540-day sim, 71%) | Fit pipeline + model |
| Validation | ~773 | Days 386–488 (18%) | Hyperparameter selection, champion selection |
| Test | ~470 | Days 489–540 (11%) | Held out — **not used for model selection** |

Split method: chronological (time-ordered, no shuffle). No future leakage possible.

Base delay rate in training data: ~36.6% (954 delayed orders on a 120-day calibration run).

## 2. Feature Pipeline

| Stage | Input | Output | Description |
|---|---|---|---|
| ColumnSelector | Raw dict (38 features) | 37-column DataFrame | Schema enforcement, NaN fill, cold-start defaults |
| InteractionFeatureAdder | 37 columns | 41 columns | 4 derived interaction features |
| ColumnTransformer | 41 features | 44 features | 6-branch preprocessing (log+scale, scale-only, binary, ordinal, count, zero-var) |

The 41-feature matrix is reproducible across training and serving: the preprocessing pipeline is serialised alongside every model artifact in MLflow.

## 8. Benchmarking Methodology Notes

- **No test set leakage**: The test split (last 11% chronologically) was held out throughout all model selection and hyperparameter tuning. All metrics above are validation-set metrics unless otherwise stated.
- **Class imbalance handling**: `scale_pos_weight` (XGBoost) and `class_weight='balanced'` (LightGBM, LogisticRegression) used throughout.
- **CV strategy**: TimeSeriesSplit (5 folds) — no shuffling, preserves temporal ordering.
- **Preprocessing per fold**: The full pipeline (ColumnSelector → InteractionFeatureAdder → ColumnTransformer) is rebuilt inside each CV fold to prevent leakage from scaling statistics.
- **Reproducibility**: All runs logged in MLflow with `mlflow.set_tracking_uri()`. Champion run IDs pinned in `backend/app/core/config.py`.

# Run SHAP analysis
jupyter notebook ml/notebooks/06_shap_analysis.ipynb

# Inspect runs in MLflow UI
mlflow ui --backend-store-uri ./mlruns --port 5000
